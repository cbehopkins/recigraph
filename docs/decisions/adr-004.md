# ADR-004: Immutable Domain Model

**Status:** Accepted

## Context

The compiler performs multiple transformation phases.

Mutable shared state increases coupling and makes compiler behaviour more difficult to reason about.

## Decision

Domain models are immutable.

Compiler phases do not modify existing objects.

Instead, each phase produces a new representation derived from the previous one.

Mutation should be avoided wherever practical.

## Consequences

Compiler phases become easier to test.

Transformations become deterministic.

Future features such as caching and incremental compilation become simpler to implement.

The resulting architecture more closely resembles modern compiler design.
