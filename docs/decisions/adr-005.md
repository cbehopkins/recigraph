# ADR-005: Incremental Compiler Development

**Status:** Accepted

## Context

The ReciGraph DSL is expected to evolve over time.

Implementing the entire language before obtaining feedback would introduce unnecessary risk.

## Decision

The compiler will be developed incrementally.

Each increment introduces a single language feature together with:

* Domain model
* Validation
* Tests
* Minimal supporting implementation

The compiler should remain in a working state after every increment.

## Consequences

Progress is continuously demonstrable.

Architectural changes can be made early when they are inexpensive.

Test coverage grows alongside the language.

Future contributors can understand the compiler by following the history of incremental additions.
