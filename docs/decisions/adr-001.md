# ADR-001: ReciGraph is a Compiler

**Status:** Accepted

## Context

The project's primary goal is to create and maintain a high-quality recipe knowledge base that can generate multiple outputs, including a website, printable cookbook, shopping lists and future AI-assisted applications.

Traditional content management approaches focus on storing and rendering documents.

This project instead authors structured data which must be validated, transformed and rendered into multiple representations.

## Decision

ReciGraph will be designed as a compiler rather than as a web application.

The compiler pipeline will consist of distinct phases including:

* Loading
* Parsing into domain models
* Validation
* Resolution
* View model generation
* Rendering

Each phase has a single responsibility and transforms immutable data into a new representation.

## Consequences

The project gains:

* Clear separation of concerns
* Strong validation before publication
* Excellent testability
* Support for multiple renderers
* A maintainable architecture as the DSL evolves

The project deliberately avoids coupling the core compiler to any particular output format.
