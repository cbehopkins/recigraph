# ADR-006: Test-Driven Development

**Status:** Accepted

## Context

The compiler performs validation and transformation of structured knowledge.

Incorrect behaviour may not become visible until much later in the compilation pipeline.

## Decision

Development follows Test-Driven Development.

Tests form part of the specification for each compiler increment.

The test suite is organised around compiler phases and includes:

* Schema tests
* Validation tests
* Resolution tests
* Renderer tests
* Acceptance tests

Golden-file testing will be used where appropriate for compiler outputs.

## Consequences

The behaviour of the compiler is specified by executable tests.

Refactoring can be performed with confidence.

The project gains a reliable regression suite as the DSL evolves.
