# ADR-003: Pydantic Models Define the DSL

**Status:** Accepted

## Context

The project requires a strongly typed representation of the DSL together with robust runtime validation.

Maintaining separate schema definitions and Python models would introduce duplication.

## Decision

The ReciGraph DSL is defined by immutable Pydantic models.

Pydantic models are the canonical schema.

No separate JSON Schema will be maintained manually.

Validation should be implemented in Pydantic wherever practical.

Cross-entity and graph validation belong to later compiler phases.

## Consequences

The project has a single authoritative definition of the language.

Type hints, runtime validation and documentation remain consistent.

Schema evolution requires changes in one place only.
